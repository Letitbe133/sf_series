Symfony Series #1

The file structure

There are 4 main directories in Symfony that we need to care about :

/config

In this directory will live all the config files necessary to run your app

Inside you will find a routes.yaml file. Take a look inside

/public

This is the only public entry point to your app.

Inside it will live all the files and assets that need to be accessed by the browser

  • Images and medias
  • Stylesheets
  • Js scripts
  • And an index.php page that will be the Front Controller of your app

The Front Controller will intercept all the requests made to your app

/src

This is your working directory, where you will write your PHP code

Inside you will find several directories, amongst them :

  • /src/Controller : which will contain your controllers classes
  • /src/Entity : which will contain your entities. Remember that entities are classes that will be used by Symfony to create tables in your database
  • /src/Migrations : which will contain the files that keep track of the migrations (or modifications) to the structure of your tables
  • /src/Repository : which will contain the repositories corresponding to your entities.
    A repositoryis a class that will help you query your tables and make SELECT requests

/templates

In this directory will live all the templates (or pages for your app)

The routing

How does it work ?

Step #1

The Front Controller of your app will intercept all incoming requests (ex /home, /about, /contact, /wtf)

Step #2

It will look for a matching route for this request, for example in the /config/routes.yaml file and will call the associated controller

Step #3

The controller that was called will send a response and display the corresponding template that it will fetch in the /templates directory

Creating your first route

#KeepItSimple

If you only render static pages (pages that will not need to display dynamic data) you won't even need to create your own controller as Symfony provides a TemplateController class that works out of the box

All you have to do is add your route in the routes config file, associate the controller and specify the name of the template to render (that can be a html, php or twig file)

More resources here

#OneStepBeyond

One other way is to create your route and specify a custom controller and method (remember a controller is a php class that has properties and methods)

Take a look in /config/routes.yaml you will find and example here

Next step is to create your controller and create inside it the specified method that will return a Response object containing the corresponding template (.html, .php, .twig)

More on this topic here

In this example, you will note that there is an alternate way to configure routes. For that you will need to install another package via composer
